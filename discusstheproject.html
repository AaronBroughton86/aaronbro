Discuss The Project

Before any project is begun, discussion must be made about wheater the project is feasible or not. If an agreement is made that the project is feasible and will be of value for the company, as well as myself, then the project can begin. A contractual agreement along with a requirments document will be made by myself, and the client, who wishes to proceed with the project.

LOT OF CONCEPTING

A requirements document containing all the features you would like to see the site being creted implemented will be written. The site will need to have all the requirments listed on the requirements document guaranteed. Failure of not gathering all the requirements could result into bad production. The project must have all the requirements of the site listed on the requirements document before a contractual agreement is signed by myself and others. Once we agree on a list of requirements then and only then can a contractual agreement take place between the client and myself. This will ensure the highest of quality of the project will be delivered without dispute between the two parties.

UI/UX PLANNING

The user interface will be implemented with the fact in mind of who your target audience is. The user interface need not be overcomplicated for your users, but simply needs to be easy and smooth sailing for your customers, and yourself. Planning the user interface is where one has to think like the user of the site themselves, and where would you expect to find things on the site. User Testing will be made to ensure that the mental model is similar of majority of the users for the site being created.

Design

The design of a project should be creative as well as visually pleasant to the eye. The desing should also not interfere with the user's main tasks. The design will be created with the fact in mind that user's don't want to get easily distracted, but they also need something to keep them intrigued in the site. As always everyone has a different vision, and I am open to see your point of views on design features of a project. The more design features there is to work with the better.
